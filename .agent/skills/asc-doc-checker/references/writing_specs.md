# 文档写作规范参考

## 一、基础语法规范

### 1.1 标点符号
- 中文描述使用中文标点，不要出现英文符号
- 标点符号要完整，除了短语，其他的语句都要加句号
- 中英文混用时英文前后不加空格
- 单位与数字之间不加空格（50m、10kg、64Kbit/s）
- 感叹号仅用于可能引发严重安全后果的警告

### 1.2 标题规范
- 使用 `# [空格][标题名]` 格式
- 标题级别逐级递增，最多四级
- 标题末尾不加标点，避免特殊字符
- 标题中不能手工添加序号（如 "## 1. 安装" 错误，应为 "## 安装"）
- 标题与正文之间空一行

## 二、Markdown 元素规范

### 2.1 列表规范
**无序列表**：使用 `*`、`+` 或 `-`，同一列表保持一致
```markdown
* 第一项
* 第二项
```
缩进 2/4 个空格。

**有序列表**：使用数字加 `.` 号
```markdown
1. 第一项
2. 第二项
```

**列表项标点**：
- 短语：不加标点
- 句子：加句号
- 混合情况：加句号

### 2.2 表格规范
- 使用标准 Markdown 表格语法，不使用 HTML 表格
- 格式：
```markdown
| 表头1 | 表头2 |
| ----- | ----- |
| 单元格 | 单元格 |
```
- 表格内换行使用 `<br>`
- 表格内列表推荐使用 `•` 表示项目符号

**反例**（HTML 表格）：
```html
<table><thead>...</thead></table>
```

**正例**（Markdown 表格）：
```markdown
| 存储单元 | 对齐要求 |
| ---------- | ---------- |
| Global Memory | 无对齐要求。 |
```

### 2.3 代码块规范
- 行内代码：`` `code` ``
- 块级代码：
```markdown
​```语言类型
代码内容
​```
```
- 上下空一行
- 缩进 2/4 个空格

### 2.4 图片规范
- 存放位置：文档同级目录的 `figures` 文件夹
- 路径：使用相对路径，正斜杠 `/`
```markdown
![alt文本](./figures/image.png)
或
![](figures/image.png)
```
- 不使用反斜杠 `\`

### 2.5 链接与锚点
**链接**：
```markdown
[《文档名称》](https://example.com)
```

**锚点**：
```markdown
# 标题文本

参考[标题文本](#标题文本)章节
```

## 三、注释符号

```markdown
> [!NOTE]说明
> 正文内容

> [!CAUTION]注意
> 正文内容

> [!WARNING]警告
> 正文内容
```

## 四、芯片过滤标签

### 4.1 标签样式
```html
<cann-filter npu-type="950">
内容（需显式说明芯片型号）
</cann-filter>
```

### 4.2 支持的芯片类型
- `950`：Ascend 950PR/Ascend 950DT
- `A3`：Atlas A3 训练/推理系列产品
- `910b`：Atlas A2 训练/推理系列产品
- `910`：Atlas 训练系列产品
- `310p`：Atlas 推理系列产品
- `310b`：Atlas 200I/500 A2 推理产品
- `x90`：Kirin X90
- `9030`：Kirin 9030

### 4.3 使用原则
- 尽量仅使用 topic、section、list、段落级别过滤
- 尽量不要使用表格按行过滤
- 标签内需要显式说明芯片型号
- 公共内容不打标签，定制内容打各自芯片标签

## 五、数据类型顺序

按位宽从小到大、有符号→无符号→浮点排序：

```
bool → int4b_t → int8_t → uint8_t → fp4x2_e2m1_t → fp4x2_e1m2_t → 
hifloat8_t → fp8_e8m0_t → fp8_e5m2_t → fp8_e4m3fn_t → int16_t → 
uint16_t → half → bfloat16_t → int32_t → uint32_t → float → 
complex32 → int64_t → uint64_t → double → complex64
```

## 六、常见错误示例

### 6.1 列表缩进错误
**反例**：
```markdown
- tensor前n个数据计算
​```cpp
template <typename T>
​```
```

**正例**：
```markdown
- tensor前n个数据计算

    ​```cpp
    template <typename T>
    ​```
```

### 6.2 HTML 表格
**反例**：使用 `<table>` 标签

**正例**：使用 Markdown 表格语法

### 6.3 图片路径错误
**反例**：`![](docs\guide\figures\image.png)`（反斜杠）

**正例**：`![](../guide/figures/image.png)`（正斜杠）
