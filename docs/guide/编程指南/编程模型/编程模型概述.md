# 编程模型概述<a name="ZH-CN_TOPIC_0000002554090609"></a>

如[异构系统](./异构系统.md)章节所述，基于昇腾处理器的应用程序通常分为两部分：**Host 代码**与**Device 代码**。其中，Host 代码运行在 CPU 上，负责设备资源管理、Host Memory与Device Memory数据搬运及任务调度等；Device 代码运行在 NPU（神经网络处理器）上，专门执行实际的计算任务。本编程指南重点讲解如何基于 **Ascend C 编程语言**编写 Device 代码，以及如何通过 Host 代码完成 Device 代码的调度与执行。

## AI Core 硬件基础

Device 上的核心计算单元为 **AI Core**（AI 处理器的计算核心）。一枚昇腾 NPU 芯片通常集成多个 AI Core，这些 AI Core 可并行工作，大幅提升整体计算吞吐量。每个 AI Core 内部结构清晰，主要包含以下核心组件：

- **标量处理单元**：负责处理控制流（如分支、循环）和地址计算，功能类似传统 CPU 核心，是 AI Core 的“控制中枢”。
- **向量处理单元**：核心执行向量运算，是 SIMD、SIMT 两种并行模型的主要载体，不同昇腾芯片架构对并行模式的支持有所差异。
- **矩阵运算单元**：专门优化矩阵乘加运算，仅兼容 SIMD 模式，是深度学习中卷积、全连接层等核心算子的“加速核心”。
- **本地存储**：AI Core内部的存储，用于存储计算所需数据，可有效降低对全局 Device Memory 的访问延迟，提升计算效率。

需要注意的是，Ascend 950PR/Ascend 950DT架构之前的 AI Core 仅支持 **SIMD** 执行模型；从 Ascend 950PR/Ascend 950DT 架构开始，AI Core 的向量计算单元同时支持 SIMD 和 SIMT 两种并行模型，为不同类型计算任务提供了更灵活的并行选择，适配更多应用场景。

> 💡 **为什么要了解 AI Core 内部结构？**  
> 理解 AI Core 的硬件组成，是编写高效 Kernel 的关键：知晓数据需显式搬运至本地存储，才能充分利用 SIMD 的高带宽优势；了解向量处理单元的功能，才能有意识地将计算逻辑向量化，最大化发挥硬件性能。

## 并行执行模型：SIMD 与 SIMT

在高性能并行编程领域，**SIMD** 与 **SIMT** 是两种核心的并行执行模型，它们定义了指令如何驱动多个计算单元协同工作，是提升程序数据吞吐量、优化计算性能的核心技术，也是 Ascend C 编程的重点。

### SIMD（单指令多数据流）

**核心概念**：SIMD（Single Instruction Multiple Data）是一种 **数据并行** 模型，核心逻辑是：一条指令在同一个时钟周期内，对多个数据元素执行完全相同的操作，实现数据的批量并行处理。

**核心特征**：
- 单指令驱动：所有并行计算单元同步执行同一条指令，操作完全一致；
- 数据同构：要求参与计算的数据类型统一、长度相同，确保指令可批量处理；
- 同步执行：所有数据的操作在同一个指令周期内完成，无独立调度逻辑，执行节奏完全统一。

**适用场景**：主要适配数据密集、操作规整、无分支或分支极少的计算任务，典型场景包括：
- 图像像素处理（如灰度化、滤波、像素缩放）；
- 音频信号分析（如降噪、信号预处理）；
- 矩阵乘法、卷积等深度学习核心运算；
- 逐元素数学函数（如向量加减、乘除、指数/对数运算）。

**SIMD [核函数Kernel](./AI-Core-SIMD编程/核函数.md)编程四步法**：SIMD 编程遵循 SPMD 模型（Single Program Multiple Data, 单程序多数据），即同一份程序运行在每个 AI Core 上，每个 AI Core 负责处理不同的数据块，具体步骤如下：
1. **Tiling（分块）设计**：将整体计算数据划分为多个大小均匀的数据块，每个 AI Core 负责处理一个数据块，实现负载均衡，避免单个 Core 过载。
2. **数据搬入**：调用 SIMD 专用 API，将计算所需的数据从 Device Memory（通常为 HBM）搬运到 AI Core 的本地缓存，减少全局内存访问延迟。
3. **数据计算**：调用向量指令，一次处理多个同构数据；需注意，数据搬运与计算过程通常是异步执行的，需插入同步指令，确保计算时数据已就绪，保证结果准确。
4. **结果搬出**：调用 SIMD 专用 API，将本地缓存中完成计算的结果，搬运回 Device Memory，供后续任务使用。

### SIMT（单指令多线程）

**核心概念**：SIMT（Single Instruction Multiple Threads）是一种 **线程并行** 模型，核心逻辑是：一条指令同时驱动多个独立线程，每个线程仅处理一个数据元素；指令同步下发，但线程可根据代码分支（如 if-else）独立执行，灵活性更高。

**核心特征**：
- 单指令控制多线程：一条指令同步调度一组线程（如 32 个线程组成的 Warp），统一发起执行指令；
- 线程独立：每个线程拥有独立的程序计数器、寄存器和执行状态，可单独处理一个数据元素，支持复杂分支逻辑；
- 硬件自动调度与掩蔽：线程的调度、切换，以及不活跃线程的掩蔽（Masking）均由硬件自动完成，无需程序员手动管理，降低开发复杂度。

**适用场景**：主要适配不规则数据访问、分支密集、稀疏计算等场景，典型场景包括：
- 深度学习中的稀疏算子（如稀疏卷积、稀疏矩阵运算）；
- 带有复杂 if-else 分支的逐元素运算；
- 具有动态数据依赖的算法（如并行前缀和、排序网络）。

**SIMT [核函数Kernel](./AI-Core-SIMT编程/核函数.md)编程四步法**：SIMT 编程同样遵循 SPMD 模型，即同一份程序运行在每个线程上，每个线程处理不同的数据元素，具体步骤如下：
1. **Tiling（分块）设计**：将整体任务拆分为多个独立线程，使线程索引与数据索引一一对应，确保每个线程处理唯一的数据元素，避免数据重复或遗漏。
2. **数据搬入**：通常通过指针直接访问 Device Memory，无需调用 SIMD 那样的专用 API；硬件会自动将所需数据从 Device Memory 加载到线程的寄存器中，简化开发流程。
3. **数据计算**：编程方式类似 CPU 标量代码，支持分支、循环等复杂控制逻辑；数据搬运与计算过程通常无需显式同步，若涉及多线程协作（如使用共享内存），需插入同步指令。
4. **结果搬出**：通常通过指针直接将计算结果写回 Device Memory，无需调用专用 API，与 CPU 编程逻辑更接近，降低上手难度。

## AI Core 编程模型

结合 AI Core 硬件特性及上述两种并行模型，Ascend C 为 AI Core 提供了三种编程路径，开发者可根据任务需求灵活选择。

### SIMD 编程（主流，全系列支持）
- **能力范围**：支持向量计算、矩阵计算，以及向量与矩阵的融合计算，覆盖深度学习大部分核心场景；
- **适用场景**：规整的、高密度的数据并行任务，如卷积、矩阵乘、逐元素变换等，是昇腾 NPU 开发的主流选择；
- **优势**：能效比高，指令执行开销低，可充分发挥硬件性能，接近 AI Core 的峰值计算能力；
- **学习路径**：详见 [AI Core SIMD编程](AI-Core-SIMD编程/)；算子开发流程参见 [SIMD算子实现](../../算子实践参考/SIMD算子实现/)。

### SIMT 编程（补充，Ascend 950PR/Ascend 950DT）
- **能力范围**：仅支持向量计算，不支持矩阵运算或向量与矩阵的融合计算，功能范围相对有限；
- **适用场景**：离散数据访问、复杂分支控制的向量算子，同时适合熟悉SIMT模型的开发者快速上手 Ascend C；
- **限制**：当前仅支持 Ascend 950PR / Ascend 950DT 芯片架构；
- **学习路径**：详见 [AI Core SIMT编程](AI-Core-SIMT编程/)；算子开发参见 [SIMT算子实现](../../算子实践参考/SIMT算子实现/)。

### SIMD与SIMT混合编程

SIMD 与 SIMT 混合编程的 Kernel，本质上仍遵循 SIMD 编程模型；在 Kernel 函数内部，可灵活调用 SIMD 或 SIMT 函数：通过 SIMT 函数处理稀疏索引、复杂分支等不规则逻辑，输出规整的数据块；再通过 SIMD 函数对规整数据块进行高吞吐量计算，兼顾灵活性与性能。详细实现方法请参考 [AI Core SIMD编程](AI-Core-SIMD编程/) 中的混合编程章节。

## AI Core 编程小结

Ascend C 通过 **SIMD + SIMT 双模执行** 的设计，在保留 SIMD 高能效、高吞吐量优势的同时，引入 SIMT 模型覆盖离散、不规则计算场景，实现了不同类型任务的高效适配。开发者在实际开发中，应根据算法的 **访存模式**（连续 vs. 离散）、**分支密度**（低 vs. 高）和 **并行粒度**，灵活选择单一编程模型或组合使用两种模型。Host 侧通过 CANN Runtime API 提供内存管理、任务调度等能力，与 Device 侧代码协同工作，共同实现异构计算的高效运行。

## AI CPU 编程模型

AI CPU 是 Device 上的辅助处理器（基于 ARM 架构），主要用于执行一些无法由 AI Core 高效处理的任务，例如：控制流复杂、数据依赖强的逻辑，或作为算子实现的 fallback 路径（当 AI Core 无法适配时的备用执行方案）。其编程模型遵循通用 CPU 编程规范，采用标准 C/C++ 语法即可开发。详细开发指南请阅读 [AI CPU编程](AI-CPU编程.md)。
