# 入门功能落地<a name="ZH-CN_TOPIC_000000_ADD_ENTRY"></a>

本节以Add（向量加法）算子为例，介绍基于Ascend C编程模型实现向量算子的基本方法，帮助开发者快速入门SIMD算子开发。

## 算子概述

Add算子实现两个向量的逐元素加法运算，计算公式为：

$$
z = x + y
$$

- x：输入，数据量为512个float元素；
- y：输入，数据量为512个float元素；
- z：输出，数据量为512个float元素；

## 算子实现

Add算子的计算逻辑遵循"搬入-计算-搬出"三段式流水结构：

1. 将输入数据x和y从Global Memory搬运到Unified Buffer；
2. 在Unified Buffer上对x和y执行向量加法操作；
3. 将计算结果从Unified Buffer搬运到Global Memory。

```cpp
constexpr uint32_t DATA_LENGTH = 512;

__aicore__ inline void Process()
{
    // 1. 分配UB内存
    AscendC::LocalMemAllocator<AscendC::Hardware::UB> ubAllocator;
    AscendC::LocalTensor<float> xLocal = ubAllocator.Alloc<float, DATA_LENGTH>();
    AscendC::LocalTensor<float> yLocal = ubAllocator.Alloc<float, DATA_LENGTH>();
    AscendC::LocalTensor<float> zLocal = ubAllocator.Alloc<float, DATA_LENGTH>();

    // 2. 搬入：GM -> UB
    AscendC::DataCopy(xLocal, xGm, DATA_LENGTH);
    AscendC::DataCopy(yLocal, yGm, DATA_LENGTH);
    AscendC::PipeBarrier<PIPE_ALL>();

    // 3. 计算：z = x + y
    AscendC::Add(zLocal, xLocal, yLocal, DATA_LENGTH);
    AscendC::PipeBarrier<PIPE_ALL>();

    // 4. 搬出：UB -> GM
    AscendC::DataCopy(zGm, zLocal, DATA_LENGTH);
    AscendC::PipeBarrier<PIPE_ALL>();
}
```

## 功能调试

### printf

该接口提供CPU域/NPU域调试场景下的格式化输出功能。

在算子kernel侧实现代码中需要输出日志信息的地方调用printf接口打印相关内容。

示例如下：

```
#include "kernel_operator.h"
AscendC::printf("fmt string %d\n", 0x123);
AscendC::PRINTF("fmt string %d\n", 0x123);
```

### DumpTensor

基于算子工程开发的算子，可以使用该接口Dump指定Tensor的内容。同时支持打印自定义的附加信息（仅支持uint32\_t数据类型的信息），比如打印当前行号等。

在算子kernel侧实现代码中需要打印Tensor数据的地方调用DumpTensor接口打印相关内容。样例如下：

```
// 3. 计算：z = x + y
AscendC::Add(zLocal, xLocal, yLocal, DATA_LENGTH);
AscendC::DumpTensor(zLocal, 1, 16);
```

打印示例如下：

```
DumpTensor: desc=1, addr=0x1000, data_type=float32, position=UB, dump_size=16
[141.015015, 124.238228, 152.882874, 96.875984, 39.287174, 45.760258, 172.856659, 158.620880, 29.610844, 92.915154, 169.371704, 129.811401, 96.124611, 118.990570, 3.930355, 58.091831]
```

## 性能调试

### msProf工具介绍
msProf工具是单算子性能分析工具。包含msprof op和msprof op simulator两种使用方式。该工具协助用户定位算子内存、算子代码以及算子指令的异常，实现全方位的算子调优。当前支持基于不同运行模式（上板或仿真）和不同文件形式（可执行文件或算子二进制.o文件）进行性能数据的采集和自动解析。

- 上板性能采集

    通过上板性能采集，可以直接测定算子昇腾AI处理器上的运行时间。该方式适合在板环境中快速定位算子性能问题。

    基于可执行文件demo通过msprof op执行算子调优：
    ```
    msprof op ./demo
    ```

    - 性能数据说明  
      命令完成后，会在默认目录下生成以“OPPROF_{timestamp}_XXX”命名的文件夹,性能数据文件夹结构示例如下：

      ```bash
      ├──dump                       # 原始的性能数据，用户无需关注
      ├──ArithmeticUtilization.csv  # cube/vector指令cycle占比
      ├──L2Cache.csv                # L2 Cache命中率，影响MTE2，建议合理规划数据搬运逻辑，增加命中率
      ├──Memory.csv                 # UB，L1和主存储器读写带宽速率
      ├──MemoryL0.csv               # L0A，L0B，和L0C读写带宽速率
      ├──MemoryUB.csv               # Vector和Scalar到UB的读写带宽速率
      ├──OpBasicInfo.csv            # 算子基础信息
      ├──PipeUtilization.csv        # 采集计算单元和搬运单元耗时和占比
      ├──ResourceConflictRatio.csv  # UB上的 bank group、bank conflict和资源冲突率在所有指令中的占比
      └──visualize_data.bin         # MindStudio Insight呈现文件
      ```

更多msProf工具使用方式请参考[MindStudio工具](https://www.hiascend.com/document/redirect/CannCommercialToolOpDev)算子调优（msProf）中的内容。

完整样例代码请参见[Add性能调优样例](https://gitcode.com/cann/asc-devkit/tree/master/examples/01_simd_cpp_api/04_best_practices/00_vector_compute_practices/add_high_performance)。
